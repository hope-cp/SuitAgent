# /evidence-review - 证据质证

## 功能说明
对新证据进行全面的质证分析，评估证据的三性（真实性、合法性、关联性），生成质证意见书和补充证据建议。

## 使用方式

### 基本用法
```bash
/evidence-review [参数]
```

### 参数选项

#### 必需参数
- `--evidence-path TEXT` - 证据文件路径（.pdf, .docx, .jpg, .png等）
- `--case-id TEXT` - 案件编号

#### 可选参数
- `--evidence-type [合同|发票|转账记录|聊天记录|邮件|录音|录像|照片|证言|鉴定报告|其他]` - 证据类型
- `--evidence-source [己方提供|对方提供|法院调取|第三方提供]` - 证据来源
- `--reviewer-name TEXT` - 质证人姓名
- `--output-dir TEXT` - 输出目录（默认：output/[案件编号]/03_证据材料/）

### 使用示例

#### 1. 合同质证
```bash
/evidence-review \
  --evidence-path "合同书.pdf" \
  --case-id "[2025]京0105民初1234号" \
  --evidence-type "合同" \
  --evidence-source "对方提供" \
  --reviewer-name "李四律师"
```

#### 2. 聊天记录质证
```bash
/evidence-review \
  --evidence-path "微信聊天记录.pdf" \
  --case-id "张三诉李四合同纠纷案" \
  --evidence-type "聊天记录" \
  --evidence-source "己方提供" \
  --reviewer-name "王五律师"
```

#### 3. 批量质证（目录）
```bash
/evidence-review \
  --evidence-path "证据材料/" \
  --case-id "[2025]京0105民初1234号" \
  --evidence-type "其他" \
  --reviewer-name "赵六律师"
```

## SubAgent 工作流

此Command会触发以下SubAgent工作流：

```
1. DocAnalyzer (文档分析器)
   ├─ 解析证据文件
   ├─ 提取关键信息
   └─ 生成结构化数据
   ↓
2. EvidenceAnalyzer (证据分析器)
   ├─ 真实性分析
   ├─ 合法性分析
   ├─ 关联性分析
   ├─ 证明力评估
   └─ 生成质证意见
   ↓
3. Researcher (法律研究者)
   ├─ 相关法条检索
   ├─ 判例研究
   └─ 适用法律分析
   ↓
4. Writer (法律文书起草器)
   ├─ 质证意见书起草
   ├─ 补充证据清单
   └─ 证据目录更新
   ↓
5. Summarizer (摘要生成器)
   ↓
6. Reporter (报告整合器)
```

## 技术实现

### 证据分析流程

1. **文档解析** (DocAnalyzer)
   - PDF文字提取和OCR识别
   - DOCX文档解析
   - 图像内容识别
   - 生成结构化数据

2. **三性分析** (EvidenceAnalyzer)
   - **真实性**：来源可靠性、内容完整性、逻辑一致性
   - **合法性**：获取方式合法、形式合法、内容合法
   - **关联性**：与争议事实的关联度、证明价值

3. **法律检索** (Researcher)
   - 相关法条检索
   - 判例研究
   - 司法解释适用

4. **文书生成** (Writer)
   - 质证意见书起草
   - 补充证据建议
   - 证据目录更新

### 质量评估标准

#### 证据等级
- **A级（强）**：三性均具备，证明力强
- **B级（中）**：三性基本具备，证明力中等
- **C级（弱）**：存在问题，证明力较弱
- **D级（无效）**：存在重大问题，无证明力

#### 评估维度
1. **来源可靠性**（25%）
2. **内容完整性**（25%）
3. **形式合法性**（20%）
4. **逻辑一致性**（20%）
5. **与争议事实关联度**（10%）

## 证据类型及注意事项

### 1. 合同类证据
**风险点**：伪造签名或盖章、合同条款不完整、日期前后矛盾

### 2. 转账记录
**风险点**：虚假转账、金额不匹配、时间逻辑错误

### 3. 聊天记录
**风险点**：截图可能被篡改、聊天记录不完整、主体身份无法确认

### 4. 邮件往来
**风险点**：邮箱可能被伪造、邮件内容被修改、发送时间存疑

### 5. 录音录像
**风险点**：录音录像被剪辑、录音环境不清楚、侵犯他人隐私

## 注意事项

1. **证据保护**：原始证据需要妥善保管，避免损坏或丢失
2. **格式支持**：支持PDF、DOCX、图片、音频、视频等多种格式
3. **批量处理**：支持目录级别批量质证
4. **质量保证**：自动进行三性检查和风险提示

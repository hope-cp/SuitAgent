# Memory 目录

## 用途说明

该目录存放SuitAgent系统的**记忆文件**，为AI提供上下文信息和知识库。包括法律知识、模板信息、最佳实践、经验积累等。

## 记忆层次

根据SuitAgent的架构，Memory系统采用三层设计：

### 1. 系统级记忆 (System-Level)
- **位置**：根目录 `claude.md` 文件
- **内容**：全局配置、系统规则、法律知识库
- **访问**：默认加载，全局共享

### 2. 个人级记忆 (Personal-Level)
- **位置**：`.claude/memory/` 目录
- **内容**：律师偏好、个性化设置、工作习惯
- **访问**：按需加载，长期保持

### 3. 案件级记忆 (Case-Level)
- **位置**：`output/[案件编号]/06_日程管理/[案件编号].md`
- **内容**：案件上下文、工作记录、时间线
- **访问**：案件处理时自动加载

## 目录结构

```
.claude/memory/
├── README.md                   # 本文件
├── legal_templates.md          # 法律模板知识库
├── official_skills_usage.md    # 技能使用说明
├── integration/                # 集成文档
│   ├── 字段对照表.md
│   ├── 工具使用指南.md
│   ├── 委托确定工作流.md
│   └── 整合工作总结.md
└── [其他记忆文件]
```

## 文件格式规范

### 命名规范
- **格式**：`{模块名}.md`
- **规则**：使用英文或中文，体现功能，无空格
- **示例**：`legal_templates.md`, `字段对照表.md`

### 文件类型

#### 1. 知识库文件
- 法律知识、操作指南、最佳实践
- 使用中文编写，便于理解
- 定期更新，保持准确性

#### 2. 配置文件
- 系统配置、参数说明、使用说明
- 格式清晰，结构化描述
- 包含版本信息

#### 3. 集成文档
- 系统整合、技术选型、决策记录
- 详细说明，便于追溯
- 包含决策理由和过程

## 内容组织原则

### 1. 模块化
每个文件专注于特定主题，避免内容混杂。

### 2. 可检索
使用清晰的标题层级，便于快速定位信息。

### 3. 版本管理
重要文件标注版本信息和更新日期。

### 4. 示例丰富
提供实际使用示例和案例。

## 添加新内容

### Step 1: 确定文件位置
- 新建文件 → 直接放在根目录
- 属于某个模块 → 创建子目录（如 `integration/`）
- 更新现有文件 → 直接编辑

### Step 2: 编写内容
1. 使用清晰的标题结构
2. 提供详细的说明和示例
3. 标注版本信息和更新日期
4. 确保内容准确和实用

### Step 3: 更新索引
如果添加了新文件或子目录，更新本README的列表。

## 最佳实践

1. **内容精炼**：避免冗长，重点突出
2. **结构清晰**：使用标题层级，便于阅读
3. **示例实用**：提供真实场景的示例
4. **定期更新**：保持信息的时效性
5. **版本标记**：重要变更标注版本号

## 注意事项

- ❌ **不要**将大型文件放在此目录（会导致上下文过大）
- ❌ **不要**存放案件相关的具体内容（应放在 `output/`）
- ❌ **不要**将实现代码放在此目录（应放在 `tools/`）
- ✅ **必须**是知识性、说明性内容
- ✅ **应该**定期清理过期信息
- ✅ **可以**使用中文编写

## 与其他目录的关系

- **与 `agents/`**：Agent配置可能引用Memory中的知识
- **与 `commands/`**：Command说明可能引用Memory中的信息
- **与 `templates/`**：模板使用说明可能引用Memory
- **与 `tools/`**：工具使用可能需要Memory中的知识
- **与 `output/`**：输出内容应引用Memory中的标准

# 📤 04 - 客户提供

## 功能定位
客户主动提供的所有材料和信息（除委托文件外）

## 关联Agent
**DocAnalyzer** - 分析客户材料

## 包含内容

### 初始材料/
- `客户初始材料清单.md` - 客户首次提供的材料清单
- `背景资料.md` - 客户背景信息
- `情况说明.md` - 客户对情况的说明
- `相关文件/` - 客户提供的原始文件
  - `合同协议/`
  - `往来函件/`
  - `证明材料/`
  - `其他文件/`

### 补充材料/
- `补充材料清单.md` - 案件过程中补充的材料
- `补充说明.md` - 客户补充的情况说明
- `新证据材料.md` - 客户新提供的证据
- `情况变化.md` - 案件情况的变化说明

### 客户反馈/
- `客户意见.md` - 客户对案件的意见和建议
- `客户确认.md` - 客户对方案和策略的确认
- `客户指示.md` - 客户的重要决策指示
- `客户满意度.md` - 客户满意度调查

### 沟通记录/
- `会议纪要.md` - 与客户的会议记录
- `电话记录.md` - 重要电话沟通记录
- `邮件往来.md` - 与客户的邮件往来记录
- `微信沟通.md` - 微信等即时通讯记录

### 需求确认书/
- `客户需求确认.md` - 客户需求的正式确认
- `服务范围确认.md` - 服务范围的明确确认
- `期望目标.md` - 客户的期望目标说明
- `特殊要求.md` - 客户的特殊要求

### 客户指示/
- `决策指示.md` - 客户的重要决策
- `授权范围.md` - 客户的授权范围调整
- `和解意向.md` - 客户的和解意向
- `上诉决定.md` - 是否上诉的决策

## 数据流转
- 为案件分析提供客户视角信息（02-DocAnalyzer）
- 为法律研究提供背景资料（03-Researcher）
- 为策略制定提供客户需求（02-Strategist）
- 为文书起草提供客户要求（06-Writer）

## 文件命名规范
- `初始材料_YYYYMMDD.md`
- `补充材料_YYYYMMDD.md`
- `客户反馈_YYYYMMDD.md`
- `沟通记录_YYYYMMDD.md`
- `会议纪要_YYYY-MM-DD.md`
- `电话记录_YYYY-MM-DD.md`

## 使用场景
1. **案件接收** - 接收和分析客户初始材料
2. **需求沟通** - 确认客户需求和期望
3. **补充信息** - 接收客户补充材料
4. **进展汇报** - 向客户汇报案件进展
5. **决策确认** - 确认客户重要决策

# 🎯 09 - 庭审笔录

## 功能定位
庭审过程记录与庭后分析

## 关联Agent
**DocAnalyzer** - 分析庭审笔录
**Strategist** - 根据庭审调整策略
**Summarizer** - 生成庭审摘要

## 包含内容

### 庭审笔录原文/
- `庭审笔录.pdf` - 原始庭审记录（法院提供）
- `庭审笔录.md` - 结构化庭审记录
- `庭审要点提取.md` - 庭审核心要点
- `当事人陈述.md` - 各方当事人陈述记录

### 庭后分析/
- `庭审情况分析.md` - 庭审整体情况分析
- `我方表现评估.md` - 我方庭审表现评估
- `对方表现分析.md` - 对方庭审表现分析
- `法官倾向分析.md` - 法官审理倾向分析

### 庭审摘要/
- `庭审摘要.md` - 庭审要点摘要（Summarizer输出）
- `争议焦点变化.md` - 庭审中争议焦点的变化
- `证据展示情况.md` - 证据展示和质证情况
- `法官询问.md` - 法官询问的要点

### 庭审策略调整/
- `策略调整建议.md` - 基于庭审的策略调整建议
- `后续行动方案.md` - 庭审后的行动方案
- `补充证据计划.md` - 庭审后补充证据计划
- `下次庭审准备.md` - 后续庭审准备事项

## 工作流程
```
庭审结束 → DocAnalyzer分析笔录 → Summarizer生成摘要 → Strategist调整策略
    ↓            ↓                ↓               ↓
  笔录获取     情况分析         要点提炼        策略优化
```

## 庭审要点分类

### 争议焦点
- **争议焦点1**：庭审中的争议和辩论
- **争议焦点2**：各方观点和证据支持
- **争议焦点3**：法官的倾向性意见

### 证据质证
- **我方证据**：对方质证意见和我方回应
- **对方证据**：我方质证意见和对方回应
- **争议证据**：存在争议的证据

### 法官询问
- **事实问题**：法官询问的事实问题
- **法律问题**：法官询问的法律问题
- **程序问题**：程序相关的询问

## 数据流转
- 触发策略优化（02-Strategist）
- 为法律研究提供新信息（03-Researcher）
- 为后续文书提供参考（06-Writer）
- 更新案件时间线（00-Scheduler）

## 输出文件示例
- `庭审笔录分析_YYYYMMDD.md`
- `庭后分析报告_YYYYMMDD.md`
- `庭审摘要_YYYYMMDD.md`
- `策略调整建议_YYYYMMDD.md`

## 使用场景
1. **庭审记录** - 整理和分析庭审笔录
2. **情况分析** - 评估庭审整体情况
3. **策略调整** - 根据庭审情况调整策略
4. **后续准备** - 准备后续庭审或程序
5. **客户汇报** - 向客户汇报庭审情况

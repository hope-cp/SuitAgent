# 📄 02 - 案件分析

## 功能定位
案件初始分析、当事人信息、诉讼请求提取的综合分析中心

## 关联Agent
**DocAnalyzer** - 案件要素提取和分析
**Strategist** - SWOT分析、策略制定

## 包含内容

### 案件分析报告/
- `案件分析报告.md` - 完整案件分析（DocAnalyzer输出）
- `案件要素提取.md` - 核心案件要素清单
- `案件事实梳理.md` - 事实时间线和逻辑关系

### 争议焦点分析/
- `争议焦点列表.md` - 识别到的争议焦点（IssueIdentifier输出）
- `焦点分析报告.md` - 争议焦点深度分析
- `焦点法理依据.md` - 争议焦点的法理基础

### SWOT分析/
- `SWOT分析.md` - 优势、劣势、机会、威胁分析（Strategist输出）
- `竞争力评估.md` - 我方竞争力分析
- `对手分析.md` - 对方当事人分析

### 诉讼策略方案/
- `诉讼策略方案.md` - 整体策略（Strategist输出）
- `应诉策略.md` - 应诉阶段策略
- `举证策略.md` - 举证和质证策略
- `庭审策略.md` - 庭审阶段策略

### 风险评估报告/
- `风险评估报告.md` - 综合风险识别与应对
- `败诉风险分析.md` - 败诉可能性及原因
- `风险应对措施.md` - 具体风险防控方案

### 当事人信息表/
- `当事人信息表.md` - 详细当事人信息
- `当事人关系图.md` - 各方当事人关系图
- `当事人联系方式.md` - 详细联系方式清单

## 数据流转
- 为后续所有Agent提供基础数据
- 为法律研究提供争议焦点
- 为策略制定提供事实基础

## 工作流程
```
DocAnalyzer分析 → IssueIdentifier识别争议焦点 → Strategist制定策略
```

## 输出文件示例
- `案件分析报告_YYYYMMDD.md`
- `争议焦点分析_YYYYMMDD.md`
- `SWOT分析_YYYYMMDD.md`
- `诉讼策略方案_YYYYMMDD.md`
- `风险评估报告_YYYYMMDD.md`
- `当事人信息表_YYYYMMDD.md`

## 使用场景
1. **案件接收** - 全面分析案件情况
2. **争议识别** - 明确争议焦点
3. **策略制定** - 制定诉讼策略
4. **风险评估** - 识别和应对风险
5. **进展汇报** - 向客户汇报分析结果

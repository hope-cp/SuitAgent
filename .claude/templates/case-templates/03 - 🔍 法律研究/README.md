# 🔍 03 - 法律研究【核心前置】

## 功能定位
争议焦点识别、法条检索、判例研究的法理支撑中心
**关键说明**：此目录在案件分析后立即进行，为策略制定和文书起草提供法理基础

## 关联Agent
**IssueIdentifier** - 争议焦点识别
**Researcher** - 法条检索、判例研究

## 包含内容

### 争议焦点分析/
- `争议焦点列表.md` - 识别到的争议焦点（IssueIdentifier输出）
- `焦点法理分析.md` - 争议焦点的法理依据
- `焦点举证责任.md` - 各焦点的举证责任分配
- `焦点裁判观点.md` - 司法实践中的裁判观点

### 法条检索/
- `民法典相关条款.md` - 民法典相关条款汇总
- `程序法相关条款.md` - 民事诉讼法相关条款
- `专门法相关条款.md` - 行业专门法律规定
- `司法解释.md` - 相关司法解释
- `法条适用路径.md` - 法条适用的逻辑路径

### 判例研究/
- `支持案例.md` - 有利判例汇总
- `不利案例.md` - 不利判例分析
- `类似案例对比.md` - 相似案例对比分析
- `案例要点提炼.md` - 案例核心要点
- `裁判规则总结.md` - 可适用的裁判规则

### 学术观点/
- `学者观点汇总.md` - 学术界的不同观点
- `理论争议.md` - 理论争议点分析
- `前沿问题.md` - 前沿法律问题研究

### 法律适用分析/
- `法律适用分析.md` - 完整的法条适用路径（Researcher输出）
- `法理依据.md` - 诉讼的法理基础
- `适用难点.md` - 法条适用的难点和争议
- `解决方案.md` - 争议问题的解决方案

## 前置意义
1. **理论基础** - 法律研究是诉讼策略和文书起草的理论基础
2. **避免返工** - 先研究法律依据，再制定策略，避免策略偏离法理
3. **一致性保障** - 确保策略和文书都基于扎实的法理分析

## 工作流程
```
案件分析(02)完成 → IssueIdentifier + Researcher并行触发 → 法律研究(03)完成
       ↓                    ↓                      ↓
   事实基础              争议焦点识别          法理基础
                                            ↓
                                        策略制定(02)
```

## 数据流转
- 为策略制定提供法理支撑（02-Strategist）
- 为证据分析提供法条依据（05-EvidenceAnalyzer）
- 为文书起草提供法理支撑（06-Writer）
- 支持增量更新（庭审后、对方提交新材料）

## 输出文件示例
- `争议焦点分析_YYYYMMDD.md`
- `法条检索报告_YYYYMMDD.md`
- `判例研究报告_YYYYMMDD.md`
- `法律适用分析_YYYYMMDD.md`
- `学术观点汇总_YYYYMMDD.md`

## 使用场景
1. **案件分析后** - 立即进行法律研究
2. **策略制定前** - 确保策略有法理依据
3. **文书起草前** - 为文书提供法理支撑
4. **庭审准备** - 强化法理基础
5. **庭审后** - 根据庭审情况更新法律研究
6. **对方提交新材料** - 更新相关法律研究
